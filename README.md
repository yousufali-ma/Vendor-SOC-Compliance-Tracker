# Third-Party Risk & SOC Compliance Tracker
**Author:** Yousuf Ali Shaik | IT Risk & Controls Analyst

### Overview
A major component of IT Audit is Third-Party Risk Management (TPRM). Companies rely on external vendors (AWS, Workday, Salesforce), and auditors must ensure these vendors have valid **SOC 1 Type 2 or SOC 2 Type 2** reports. Furthermore, auditors must ensure that **Complementary User Entity Controls (CUECs)** are actively mapped and tested internally.

### What This Script Does
This automated audit tool:
1. Ingests a vendor database.
2. Identifies any vendors whose SOC reports have expired prior to the current audit period.
3. Flags vendors where CUECs have not been mapped to internal control frameworks.
4. Generates a deficiency report to be escalated to the Vendor Management Office (VMO) for remediation or bridge letters.

**Skills Demonstrated:** Vendor Risk Management, SOC 1 / SOC 2 Analysis, CUEC Mapping, Deficiency Tracking.
